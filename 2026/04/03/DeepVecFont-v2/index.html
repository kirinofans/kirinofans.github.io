[论文笔记] DeepVecFont-v2: Transformer驱动的矢量字体生成

paper:DeepVecFont-v2: Exploiting Transformers to Synthesize Vector Fonts with Higher Quality
Comments: Accepted by CVPR 2023. Code: this https URL
Subjects: Computer Vision and Pattern Recognition (cs.CV); Graphics (cs.GR)

摘要

矢量字体以 SVG 为标准格式，具备无损缩放、可编辑、文件体积小的特性，是现代数字设计的基础设施。然而，高质量矢量字体设计门槛高、成本高，尤其是字符集庞大的中文。现有方法中，栅格生成方法无法输出可商用矢量格式；早期 RNN 方案存在误差累积、笔画断裂等问题；前代 SOTA 方法 DeepVecFont 受限于 RNN 的串行计算瓶颈，且图像引导后处理易引入伪影。

本文提出了 DeepVecFont-v2，核心改进是将DeepVecFont第一代的 RNN 替换为 Transformer，提升对长序列、复杂拓扑字符的建模能力。通过设计适配 Transformer 的矢量轮廓表示方法、贝塞尔曲线对齐损失以及上下文自精炼模块，该方法摆脱了对图像引导后处理的强依赖，在中英文矢量字体生成任务上全面超越现有 SOTA 方法。

1. 研究背景与目的

1.1 矢量字体的核心应用价值

矢量字体以 SVG 为标准格式，具备无损缩放、可编辑、文件体积小的特性，广泛应用于文档排版、品牌视觉系统、动态媒体内容生产等专业设计场景。

1.2 人工设计的高门槛与成本

高质量矢量字体需设计师逐字符精确控制贝塞尔曲线，周期长、成本高。中文等字符集庞大、拓扑复杂的文字体系，设计工作量呈指数级增长，自动化需求迫切。

1.3 现有技术的缺陷

  1. 栅格生成方法：仅能输出像素图像，无法提供可商用矢量格式，放大后易出现锯齿、失真。
  2. 早期 RNN 方案：易出现误差累积，生成的字符存在轮廓错位、笔画断裂、结构失真等问题。
  3. 前代 SOTA 方法 的固有缺陷：
    • RNN 序列建模的结构性瓶颈：采用 RNN 进行序列建模，循环结构的串行计算特性导致长序列处理能力受限，难以有效捕捉复杂字符的全局拓扑依赖关系，复杂字形生成稳定性不足。
    • 图像引导后处理的伪影问题：通过对齐光栅化结果和合成图像来微调原始矢量字形的坐标，该设计易过拟合图像噪声，引入角点过平滑、连接处欠平滑、笔画自相交等伪影，生成结果无法直接商用。

1.4 本文目的

提出 DeepVecFont-v2，实现以下目标：

  1. 替换 RNN 为 Transformer，提升模型对长序列、复杂拓扑字符的建模与生成能力。
  2. 设计适配 Transformer 的矢量轮廓表示方法，摆脱对图像引导后处理的强依赖，解决序列生成中的误差累积问题。
  3. 消除轮廓失真、毛刺、错位等问题，在中英文矢量字体生成任务上全面超越现有 SOTA 方法，实现少样本场景下的高保真风格迁移。

2. 相关工作

2.1 栅格图像字体合成

这类方法以像素级的字形图像为生成目标，核心基于 VAE、GAN 等深度生成模型实现字体风格迁移与合成，是字体生成领域的早期主流研究方向。

  • 局限：无法输出可编辑、无损缩放的矢量格式，放大后易出现锯齿、失真问题。

2.2 矢量字体生成

  • 传统非深度学习方法：基于规则或模板。
  • 深度学习驱动的方法：RNN/LSTM 序列建模（如 Easyfont）。
  • DeepVecFont (v1)：首次采用图像与序列双模态学习框架。通过 CNN 图像编码器与双向 LSTM 序列编码器分别提取特征，融合得到统一的字体风格隐向量，利用反卷积图像解码器与 LSTM+MDN 生成栅格图像与矢量绘图指令，最后经过图像引导的轮廓精修后处理优化。虽然效果不错，但仍受限于 RNN 结构和后处理伪影。

3. 方法架构

3.1 SVG Embedding (矢量嵌入)

目标：把指令、坐标、字形大小、位置转成模型可处理的向量。

定义：本文把一个矢量字定义成一串绘图指令序列 Gi=[Ci,1,Ci,2,...,Ci,Nc]G_i = [C_{i,1}, C_{i,2}, ..., C_{i,N_c}]Gi​=[Ci,1​,Ci,2​,...,Ci,Nc​​]。每条指令 Ci,jC_{i,j}Ci,j​ 固定写成：Ci,j=(zi,j,pi,j)C_{i,j} = (z_{i,j}, p_{i,j})Ci,j​=(zi,j​,pi,j​)。

  • zi,jz_{i,j}zi,j​：指令类型（4 种之一）：MoveFromTo（移动），LineFromTo（直线），CurveFromTo（三阶贝塞尔曲线），EOS（结束）。
  • pi,jp_{i,j}pi,j​：坐标点集合（控制点）。

关键改进：松弛表示

  • 问题：传统 SVG 相邻指令共用起点和终点（前一个终点 = 后一个起点），这会导致前一个指令画错从而后面全部跟着歪，不适合 Transformer（注意力需要完整位置信息）。
  • 本文规定：每条指令自己单独拥有起点和终点，且全部指令固定用 4 个坐标点，EOS 不用坐标。
  • 约束与推理：训练时加约束强制前一条终点 ≈\approx≈ 后一条起点；推理时把这两个点取平均合并。

向量构成：
每条指令 Ci,jC_{i,j}Ci,j​ 变成一个 dEd_EdE​ 维向量 ei,je_{i,j}ei,j​，由 4 部分相加得到：

ei,j=cmd+args+wh+pose_{i,j} = \text{cmd} + \text{args} + \text{wh} + \text{pos} ei,j​=cmd+args+wh+pos

  • cmd：指令类型嵌入。
  • args：坐标参数嵌入。
  • wh：字形高宽嵌入。
  • pos：Transformer 绝对位置编码。

3.2 编码器

继承了前代 DeepVecFont 的双模态（图像 + 序列）思想，但全部换成 Transformer 架构。编码器由两个分支组成：

  1. 图像编码器：普通 CNN，输出图像风格特征 fimgf_{img}fimg​。
  2. 序列编码器：6 层 Transformer Encoder，输出序列结构特征 fseqf_{seq}fseq​。

3.3 模态融合

  • 把 fseq0f^0_{seq}fseq0​ 与 fimgf_{img}fimg​ 拼接得到融合特征 fff。
  • 使用重参数化技巧，让 fff 服从高斯分布，保证生成稳定。

3.4 解码器

解码器分两条路，同时生成图像和矢量指令。

  1. 图像解码器：反卷积 CNN，输出目标字的栅格图像 I^t\hat{I}_tI^t​。
  2. 序列解码器：Transformer Decoder。
    • 输入：融合特征 fff 和序列风格特征。
    • 输出：指令类型 z^t\hat{z}_tz^t​（Move/Line/Curve/EOS）、坐标点 p^t\hat{p}_tp^​t​（x1,y1…x4,y4x_1, y_1 \dots x_4, y_4x1​,y1​…x4​,y4​）。

与 DeepVecFont 的区别：

  1. 序列编解码全部换成 Transformer 而不是 LSTM，Transformer 用自注意力注意所有指令。
  2. DeepVecFont 必须靠图像引导后处理，v2 靠特征融合与自精炼。

3.5 Context-based Self-refinement (基于上下文的自精炼)

原因：在推理阶段，Transformer 遵循 RNN 的序列生成流程，以前序预测结果为条件预测后续步骤，会产生误差累积，导致字形失真。而字形本身有很强的几何先验（如对称性、笔画宽度的平滑变化），这些上下文信息可修正生成结果。

结构：2-layer Transformer decoder。

  • 作用：再编码初始序列，利用全局上下文做修正，输出更准确的指令与坐标。
  • 输入：
    • QQQ：初始生成序列的嵌入。
    • K,VK, VK,V：前序序列解码器的记忆的线性投影。
    • MMM：Mask 掩码（0 或负无限大），让模型关注有效长度的指令，忽略后面的空位置。

3.6 损失函数


模型训练包含以下损失函数：

  1. LimgL_{img}Limg​：图像重建损失（L1 + 感知损失）。
  2. LCE_initL_{CE\_init}LCE_init​：初始序列交叉熵损失。
  3. LCE_refineL_{CE\_refine}LCE_refine​：精炼序列交叉熵损失。
  4. LconsL_{cons}Lcons​：一致性约束损失。计算相邻指令的起点与终点的坐标差得到，保证笔画连续不断开。
  5. LbezierL_{bezier}Lbezier​：贝塞尔曲线对齐损失。除了监督贝塞尔曲线的 4 个控制点，额外在曲线上采样 0.25、0.5、0.75 三个辅助点，计算生成曲线与真实曲线在采样点的坐标差。
  6. LklL_{kl}Lkl​：KL 散度损失，重参数化操作时约束隐空间服从高斯分布。

4. 实验

4.1 实验细节

数据集：

  • 英文：沿用 DeepVecFont 数据集，8035 个训练字体、1425 个测试字体。
  • 中文：自己构建，212 个训练、34 个测试；只选 52 个结构简单的汉字，并用仿射变换把训练集扩大 10 倍。


参数情况：
贝塞尔曲线采样点数：0 个 →\to→ 3 个效果大幅提升；3 个 →\to→ 9 个提升很小。

4.2 实验结果

  • 生成的少样本中文字体向量结果对比显示，DeepVecFont-v2 能够生成结构准确、线条流畅的矢量字形。
  • 中英文矢量字体插值结果平滑，权重、宽度和样式变化自然。

4.3 消融实验


基线：把 DeepVecFont 的 LSTM 简单换成 Transformer。
逐步加入 3 大创新模块的效果：

  1. 松弛表示：保证指令连接顺畅、不崩坏。
  2. 贝塞尔损失：消除曲线扭曲、更贴合真实轮廓。
  3. 自精炼：去掉毛刺、伪影，得到紧凑、平滑、高质量字形。

5. 结论和局限

5.1 结论

本文提出了全新的矢量字体生成模型 DeepVecFont-v2，核心改进是将前代 DeepVecFont 中用于序列建模的 RNN 网络全面替换为 Transformer 架构，并针对性设计了三大关键创新模块：

  1. 松弛表示：适配 Transformer 全局注意力机制。
  2. 贝塞尔曲线对齐损失：通过采样辅助点提升曲线拟合精度，解决仅监督控制点导致的轮廓贴合度不足问题。
  3. 上下文自精炼模块：摆脱对图像引导后处理的依赖。

实现了端到端直接生成高质量矢量字体，无需额外后处理。实验结果充分证明，DeepVecFont-v2 在英文、中文矢量字体生成任务上，性能全面超越现有 SOTA 方法，生成效果更精准、无伪影，对复杂拓扑字符的适配性大幅提升。

5.3 局限性

  1. 超长序列复杂字形生成失败：对于包含大量长绘图路径的超复杂汉字（如“霸”），模型无法生成正确结果。
  2. 拓扑结构变化场景建模困难：同一中文字符在不同字体风格下可能出现大幅拓扑结构变化，模型难以学习到稳定的 SVG 嵌入表示，导致生成效果不稳定。
