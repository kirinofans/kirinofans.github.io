[论文笔记] 基于 Transformer 的中文字符 SVG 矢量字体生成

paper:SVG Vector Font Generation for Chinese Characters with Transformer
Comments: Accepted to ICIP 2022
Subjects: Computer Vision and Pattern Recognition (cs.CV)

摘要

中文字体设计因其字符数量巨大、结构复杂，一直是一项耗时费力的工作。现有的字体生成研究多集中于光栅图像，而矢量字体生成研究相对匮乏，且多针对英文等简单字形。

本文提出了一种基于 Transformer 的全新网络架构，首次实现了仅通过单一样式参考和单一内容参考，自动生成符合样式的中文 SVG 矢量字体。该方法通过分层 Transformer 结构与改进的 AdaIN 风格融合机制，有效捕捉了矢量图形的空间排布和形状特征，填补了中文矢量字体自动化生成的空白。

1. 研究背景与挑战

1.1 中文字体设计的痛点

中文字体人工设计劳动强度大、周期长，对自动化生成技术需求迫切。SVG 作为矢量图形的标准格式，非常适合字体表示，但在生成任务中面临显著挑战。

1.2 矢量图形生成的难点

与自然语言处理不同，矢量图形的生成具有独特的困难：

  1. 表示非唯一性：即便渲染结果外观相同，其底层的指令表示形式也可能完全不同。
  2. 特征捕捉困难：难以从一系列离散指令中捕捉空间布局和形状特征。
  3. 优化难度大：矢量图形到光栅图像的渲染过程通常不可微，难以直接整合到神经网络中进行端到端优化。现有的可微渲染方法往往效率低下。

1.3 现有研究的局限

  • 光栅图像生成：现有中文字体生成多聚焦于此，无法直接用于矢量字体生产。
  • 矢量生成方法：
    • 多针对简单英文字母。
    • 需大量用户干预或拓扑约束，无法适配中文字形的复杂性。
    • 基于深度学习的方法（如 RNN、部分 Transformer）存在特征捕捉不足、长序列生成效果差的问题。

1.4 本文研究目的

  1. 提出新架构：基于 Transformer 的网络，实现 “样式/内容参考 -> 风格迁移 -> 中文SVG矢量字体” 的自动生成。
  2. 设计专属模块：适配中文字形的网络结构及损失函数（特别是倒角损失）。
  3. 构建数据集：包含 66 种无衬线字体、800 个中文字符的专属矢量数据集。

2. 相关工作

2.1 基于组件拼接的方法

  • Automatic generation of chinese vector fonts via deep layout inferring
  • 原理：模拟人类学写字，拆解笔画/部件 -> 学习布局规律 -> 拼装预测。
  • 局限：依赖拼接部件，对于复杂或连笔字形效果有限。

2.2 基于骨架生成的方法

  • Zhang et al. (2017) & Tang et al. (2019) —— FontRNN
  • 原理：基于深度学习生成骨架点序列。
  • 局限：只生成骨架点，并非真正的 SVG 矢量轮廓，无法直接使用。

2.3 DeepSVG

  • DeepSVG: A Hierarchical Generative Network for Vector Graphics Animation
  • 原理：分层生成（顶层结构 -> 路径属性 -> 贝塞尔曲线优化），采用非自回归方式并行预测。
  • 局限：对于结构极其复杂、序列过长的中文字符，无法有效生成。

3. 模型架构

本文提出的方法核心在于分层 Transformer 结构，主要分为输入表示、编码器和解码器三个部分。

3.1 SVG 表示与输入处理

为了适配模型输入，论文对 SVG 进行了层级化拆解和向量化：

  • 层级拆解：图形 →\to→ 路径 →\to→ 命令。
    • 图形：单个中文字符的完整 SVG。
    • 路径：每个图形由 NPN_PNP​ 个路径组成，每个路径带有可见性 vi∈{0,1}v_i \in \{0, 1\}vi​∈{0,1}。
    • 命令：每个路径由 NCN_CNC​ 个绘制命令组成。仅使用 4 类核心命令（M 移动、L 直线、C 三次贝塞尔曲线、Z 闭合路径），并增加 <SOS>/<EOS> 标记。
  • 命令向量化：
    • 每个命令表示为 命令类型 + 6维坐标参数。固定长度，缺省参数填 -1。
    • 嵌入：将命令转换为 256 维嵌入向量 dE=256d_E=256dE​=256。
    • 索引嵌入：加入可学习的位置信息，区分不同路径和命令的位置。

最终输出为样式参考 SVG 和内容参考 SVG 的路径序列嵌入向量（维度：12 paths×100 commands×25612 \text{ paths} \times 100 \text{ commands} \times 25612 paths×100 commands×256）。

3.2 编码器

编码器旨在提取样式参考 SVG 的全局风格特征 zSz_SzS​。采用分层结构，去掉了 DeepSVG 中的 VAE 结构。

  1. 路径级编码器 E(1)E^{(1)}E(1)：
    • 输入：单个路径的 NCN_CNC​ 个嵌入向量。
    • 处理：通过 6 个 Transformer 块捕捉路径内命令关联，经平均池化得到路径特征向量 uiu_iui​。
  2. 图形级编码器 E(2)E^{(2)}E(2)：
    • 输入：NPN_PNP​ 个路径特征向量 uiu_iui​（加索引嵌入）。
    • 处理：通过 6 个 Transformer 块捕捉路径间全局关联，平均池化后输出 256 维全局风格特征向量 zSz_SzS​。

特点：分层处理（先命令后路径），全程 Transformer + 池化，无循环结构。

3.3 解码器

解码器旨在保留内容特征并融合风格特征，生成新的 SVG。采用分层结构并嵌入改进型 AdaIN 风格融合机制。

  1. 预处理：对内容参考 SVG 的每个路径嵌入向量进行平均池化，得到路径级内容嵌入 eˉi(C)\bar{e}^{(C)}_ieˉi(C)​。
  2. 图形级解码器 D(2)D^{(2)}D(2)：
    • 目标：提取全局内容特征，判断路径可见性。
    • 过程：输入内容嵌入，经带 AdaIN 的 Transformer 块融合 zSz_SzS​。
    • 输出：
      • 分支 1：路径全局特征向量 wiw_iwi​。
      • 分支 2：通过 MLP 预测路径可见性 v^i\hat{v}_iv^i​（二分类）。
  3. 路径级解码器 D(1)D^{(1)}D(1)：
    • 目标：生成具体绘制命令。
    • 输入：内容参考路径嵌入 + 路径全局特征 wiw_iwi​ + 全局风格 zSz_SzS​。
    • 过程：
      • wiw_iwi​ 通过仿射变换拼接到 Transformer 中间层。
      • zSz_SzS​ 通过 AdaIN 融合。
    • 输出：通过两个 MLP 头分别预测命令类型 c^\hat{c}c^（多分类）和坐标参数 X^\hat{X}X^（回归）。

3.4 损失函数

总损失函数由四部分组成：

Ltotal=Lvis+Lcmd+Larg+LcfrL_{total} = L_{vis} + L_{cmd} + L_{arg} + L_{cfr} Ltotal​=Lvis​+Lcmd​+Larg​+Lcfr​

  1. 可见性损失 LvisL_{vis}Lvis​：交叉熵损失，用于预测路径是否可见。
  2. 命令类型损失 LcmdL_{cmd}Lcmd​：交叉熵损失，用于预测命令类型。
  3. 参数损失 LargL_{arg}Larg​：L1 损失，计算预测坐标与真实坐标的绝对值差。
  4. 倒角损失 LcfrL_{cfr}Lcfr​ (核心创新)：
    • 对真实路径和预测路径采样点云。
    • 计算两个点云之间的倒角距离。
    • 作用：解决了坐标参数表示非唯一的问题，直接优化视觉外观的相似度。

4. 实验分析

4.1 实验设置

  • 数据集：Fontworks 商用字体 + 网络免费字体。
  • 范围：无衬线字体，共 66 种风格，800 个汉字。
  • 划分：59 种字体训练，7 种字体测试。
  • 超参数：训练 1500 轮，最大路径数 NP=12N_P=12NP​=12，每路径最大命令数 NC=100N_C=100NC​=100。

4.2 实验结果

  1. 方法有效性：提出的分层 Transformer + AdaIN + 倒角损失架构，能有效捕捉矢量图形的空间结构。
  2. 对比优势：现有 SOTA 方法（DeepSVG、DeepVecFont）无法生成复杂的中文字符，本文方法首次成功实现了仅凭单一样式参考生成中文矢量字体。
  3. 生成质量：在无衬线（黑体类）中文字体上验证了方法的有效性和优越性。


5. 结论

本文针对中文字体设计的痛点，提出了一套基于 Transformer 的 SVG 矢量字体生成框架。其核心贡献在于：

  1. 首次定义并解决了 “从样式/内容参考生成中文 SVG 矢量字体” 的问题。
  2. 设计了分层 Transformer 架构与适配的损失函数，有效处理了矢量图形生成中的长序列和空间结构问题。
  3. 构建了首个相关数据集，实验证明了该方法在生成复杂中文字形方面的显著优势。
