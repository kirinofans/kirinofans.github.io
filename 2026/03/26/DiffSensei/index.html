[论文笔记] DiffSensei: 让漫画角色听剧情指挥

paper:DiffSensei: Bridging Multi-Modal LLMs and Diffusion Models for Customized Manga Generation
Comments: [CVPR 2025] The project page is this https URL
Subjects: Computer Vision and Pattern Recognition (cs.CV)

摘要


故事可视化旨在根据文本描述生成视觉叙事，其中漫画生成对角色一致性、多角色布局控制及对话框融合提出了更高要求。现有方法多集中于风格迁移，难以从零开始生成定制化内容，且面临角色僵硬、布局控制粗糙等问题。

本文提出 DiffSensei，一个用于定制化漫画生成的创新框架。该框架首次将扩散模型与多模态大语言模型（MLLM）相结合，实现了角色动态控制、精确布局控制及对话框控制。同时，构建了首个大规模多角色漫画生成数据集 MangaZero。实验表明，DiffSensei 在角色一致性、布局准确性及文本响应度上均显著优于现有方法。

1. 研究背景与目的

1.1 漫画生成的独特挑战

漫画作为一种特殊的故事可视化形式，其生成任务面临独特挑战：

  1. 角色一致性：要求角色在多格画面中保持外观一致。
  2. 多角色布局：需精确控制多角色的空间位置。
  3. 对话框融合：需将对话框无缝融入叙事画面。

1.2 现有方法的局限

  • 任务偏差：当前研究多集中于将普通图像转换为漫画风格，而非根据文本生成定制化内容。
  • 角色僵硬：零样本角色定制方法常导致“复制粘贴”效果，缺乏表情和动作多样性。
  • 控制缺失：难以跨场景定制角色，缺乏对布局和对话框的精细控制。

1.3 本文目的

  1. 提出新框架：DiffSensei，结合扩散模型与 MLLM，解决动态角色控制与布局控制问题。
  2. 构建新数据集：MangaZero，填补多角色、多状态漫画生成数据集的空白。

2. 相关工作

2.1 故事可视化

  • 现状：主流依托扩散模型（如 StoryDiffusion），利用 IP-Adapter 等技术实现多角色控制。
  • 局限：现有数据集缺乏角色 ID 和表情标注，导致跨场景一致性差；不支持多角色位置精准控制；未考虑对话框布局。

2.2 漫画生成

  • 现状：主流研究集中于风格转换任务或理解任务（如 Magi 的文本转录、角色检测）。
  • 局限：无法基于文本和参考图从头生成包含连贯剧情的定制化漫画。

2.3 MLLMs for Personalized Image Generation

  • 现状：LLaVA 等模型实现了文本与图像特征的对齐。
  • 局限：难以处理多角色关系建模和群体动作协调，缺乏对连续帧中角色状态一致性的支持。

3. DiffSensei 方法框架

3.1 数据集构建：MangaZero


为解决数据缺失问题，构建了包含 43,264 张漫画页、427,147 个漫画格的大规模数据集。

  • 数据来源：选取 48 部漫画作品。
  • 标注流程：
    1. 使用 Magi 模型自动标注画框、角色边界框、角色 ID 及对话框。
    2. 使用 LLaVA-v1.6-34B 为每个画格生成说明文字。
    3. 人工修正机器标签。
  • 划分：96 页评估集，43,168 页训练集。

3.2 方法概览

DiffSensei 采用两阶段训练策略，输入为文本提示、角色参考图、布局信息，输出为定制化漫画格。

Stage 1：布局控制与角色特征提取

目标：训练具有布局控制的多角色定制图像生成模型。

  1. 角色特征提取：
    • 利用 CLIP 和 Magi 模型分别提取局部特征和漫画风格特征。
    • 通过重采样器融合筛选，输出统一维度的角色特征 cic_ici​。
  2. 掩码交叉注意力：
    • 控制每个角色仅在指定的边界框 BicB_i^cBic​ 内生成，避免多角色位置混乱。
  3. 对话框布局编码：
    • 控制对话框的位置 BidB_i^dBid​ 和形状（不生成文字内容）。

通过扩散损失优化 U-Net，确保生成内容符合角色外观、布局要求和漫画风格。

Stage 2：MLLM 适配器训练

目标：根据文本提示调整角色的表情、动作、姿态，同时保留核心外观。

  • 输入：冻结的 Stage 1 生成器、源角色特征、文本提示 TTT。
  • 可训练参数：MLLM 的 LoRA 权重、输入/输出重采样器。
  • 损失函数：

    Ltotal=λlm⋅Llm+λmse⋅Lmse+λdiff⋅LdiffL_{total} = \lambda_{lm} \cdot L_{lm} + \lambda_{mse} \cdot L_{mse} + \lambda_{diff} \cdot L_{diff} Ltotal​=λlm​⋅Llm​+λmse​⋅Lmse​+λdiff​⋅Ldiff​

    • LlmL_{lm}Llm​：约束特殊令牌格式，确保跨模态交互正常。
    • LmseL_{mse}Lmse​：约束角色特征，精准响应文本提示内容。
    • LdiffL_{diff}Ldiff​：约束特征兼容性。

4. 实验结果与分析


4.1 定性对比

  • SEED-Story：叙事混乱，故事不连贯。
  • StoryDiffusion：适配复杂布局能力弱。
  • MS-Diffusion：角色僵硬，缺乏动态变化。
  • DiffSensei：精准还原文本细节，角色一致性强，布局合理。

4.2 消融实验


通过逐一移除核心模块验证其有效性：

  • 角色掩码注意力注入：有效提升角色一致性（DINO-C 分数提升）。
  • 对话框掩码编码：对话框位置 F1 分数升至 0.653，是控制布局的关键。
  • Magi 图像编码器：显著提升 DINO-I、DINO-C 和 F1 分数，证明其对漫画特征提取的重要性。
  • MLLM 适配器：使角色精准响应文本提示，同时保持外观一致。

评价指标：

  • FID（图像真实度）
  • CLIP（文本-图像匹配度）
  • DINO-I（整体风格一致性）
  • DINO-C（角色一致性）
  • F1 Score（对话框位置准确率）

5. 结论与局限

5.1 结论

DiffSensei 是首个专为多角色定制化故事可视化设计的框架。其核心贡献包括：

  1. 掩码注意力控制：实现对角色布局的精准管理。
  2. 对话框布局编码：适配漫画场景，精准控制对话框位置。
  3. MLLM-based 特征适配器：实现角色灵活响应文本提示，动态调整表情动作。
    该方法填补了定制化漫画生成的研究空白。

5.2 局限性

  1. 非端到端文字生成：仅控制对话框布局，不生成内容文字，需人工后期编辑。
  2. 风格限制：目前聚焦于黑白漫画生成，未涉及彩色漫画。
  3. 资源依赖：依赖大规模数据集，对算力要求较高。
